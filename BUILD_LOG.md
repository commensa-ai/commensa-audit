## Session 7 notes (0.3.0) — extraction limiting

**Why:** the OSS-sweep protocol needs to audit large repos without walking their whole history, and an unbounded default is a footgun on a 10k-PR repo. Promoted from a one-off big-repo proof to a pre-launch release.

**Flags (both optional, newest-first):** `--since YYYY-MM-DD` (inclusive UTC date on created_at) and `--max-prs N` (keep the N newest; **default 500**, `0` = no cap).

**Architecture (matches GitHubExtractor's shape so engine reuse is mechanical in D-B):**
- `commensa_audit/extractors/local_clone.py` — `LocalCloneExtractor(repo_dir)`. Subprocess `git log` with a sentinel-delimited `--pretty=format:` string + `--numstat`. ## Session 9 notes (Phase D-B) — engine wiring, encoding hardening, cross-mode gate

**Encoding hardening — the D-A review finding was real and silent.** Empirically verified the four `git log --numstat` variants against a synthetic repo with `café/file.txt`:
